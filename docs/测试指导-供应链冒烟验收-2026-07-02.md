## 阶段六　库存 · 可用量核对（★本次最关键 · 角色：采购/仓库）

> 这一阶段专门验证上周修的 **color key 问题**：带颜色的物料，收货后库存/可用量要能正确对上。

### 步骤 6.1　看库存可用量
- **在哪**：`采购 / 供应链` → `📦 库存`（`/procurement/inventory`）
- **预期**：看到 `测试面料-蓝` 一行，各列：
  - **在库** = `500`（刚收的）
  - **预留** = `0`（正常，预留要到后续阶段才自动建）
  - **可用** = `400`（= 在库 500 − 安全库存 100；★若可用显示 500 说明安全库存没生效，记下来）
  - **缺口** = 空
- **结果**：在库＿＿ 预留＿＿ 可用＿＿ ｜ ☐ ✅ / ❌ ＿＿＿＿＿＿

> ★ **核心判定**：如果这里**看到了 `测试面料-蓝`（带颜色）且在库=500**，说明 color key 修复**生效**。
> 如果**这一行不出现 / 在库为 0 / 物料名不对**，立刻记下来——这正是我们要验的点。

### 步骤 6.2　看流水明细
- **点**该行 `明细`
- **预期**：展开看到一条 `入库 +500`，类型/时间/来源都在

## 附：本次"是设计如此、不是 bug"的清单（避免误报）

| 现象 | 说明 |
|---|---|
| 预留列多为 0 | 自动预留要到后续版本(SC-P4)才接;现在预留是手动能力,没人建就是 0 |
| 出厂日/订单类型要手填 | 运营字段本就不在报价/PO 里,系统只预填上次值 |
| 采购内核/"下一步建议"面板暂时没有 | 计算后端已就绪,面板(Execution Copilot)还没做,不在本次测试范围 |
| 安全库存只对设了策略的物料生效 | 没设 `安全库存` 的物料,可用 = 在库 − 预留 |
| 老数据(修复前建的库存)可能对不上颜色 | color key 修复只对**新流水**生效,请用本指导新建的数据验 |
