# 节点改期 · 使用说明（给业务 / 采购 / 生产）

> 一句话:节点红了(逾期)不用干瞪着——**责任人申请改个新完成日期,相关部门点头,红就消,改成"受控中"**。

---

## 一、什么时候用

- 你负责的**节点逾期变红**了(比如"采购单下达"过了日子还没下)。
- 采购风险中心出现**红色风险**(比如"供应商交期风险")。
- 原因往往不在你:客户没确认尺寸表、供应商没回预计到货日、上游没交……

**别让它一直红。** 红=没人管;申请改期后=有人签字在管。

---

## 二、怎么申请改期(责任人)

1. 打开那张订单 → 找到红的节点 → 点「**申请延期 / 改期**」。
2. 填两样:
   - **新完成日期**(你估计什么时候能做完 / 料能到)
   - **原因**(比如"客户尺寸表未确认,预计 X 号确认")
3. 提交 → 系统自动发给**该管的人**确认(见下表)。

---

## 三、谁来确认(审批链)

系统按"谁延期"自动找人,**逐级点头,全点完才生效**:

| 谁延期 | 依次确认 |
|---|---|
| **业务执行** 延期 | 业务开发 |
| **采购** 延期 | 业务执行 → 业务执行经理 |
| **生产** 延期 | 业务执行 |
| 其他 | 管理员 |

- 审批人打开申请,能看到一条**进度条**:谁签了(✓绿)、轮到谁(⏳紫)、还没到谁(灰)。
- **只有轮到你**的时候你点确认才有效;没轮到你点了会提示"不是你这步"。
- 逐级点完 → **红点消失**,节点日期更新,系统自动把后面节点重排一遍。

---

## 四、退交期 还是 转紧急?(关键决策)

如果这个改期**会影响给客户的整体交期**,最后确认时要**二选一**:

| 选项 | 意思 | 后果 |
|---|---|---|
| 📆 **退交期** | 整体交期往后推 | 系统把最终交期和后面节点都顺延。**等于跟客户说要晚**——业务开发要对客户负责 |
| 🚨 **转紧急** | 交期不变,压缩后面 | 系统给订单打「**交期紧急**」标,并**追加 采购+生产 确认**他们能赶。两方点头才生效 |

**简单判断**:能跟客户商量晚几天 → 退交期;客户交期不能动 → 转紧急(那后面采购、生产就得赶)。

---

## 五、采购风险:填预计到货日(采购专用)

采购风险中心里"供应商交期风险 · 预计**未定**"的红,多半是**没填预计到货日**:

1. 风险卡上点「**📅 填预计到货日**」→ 填供应商给你的到货日 → 保存。
2. 结果:
   - 到货日 **不晚于**需求日 → 下次刷新(≤15 分钟)**红自动消**。
   - 到货日 **晚于**需求日 → 显示"预计晚到 X"(不再是"未定"),这时候要么**催货**,要么走上面的**改期申请**。

> 同一个料同一个供应商的多个颜色,现在只报一条,不重复。

---

## 六、常见问题

- **Q:我点了确认,怎么没生效?**
  A:多级链要**每一级都点完**才生效。看进度条,可能还轮到别人。
- **Q:红消了会怎样?**
  A:节点日期更新成新日期,变"受控中",不再算逾期风险;全程谁申请、谁批留了痕。
- **Q:转紧急后采购/生产不确认?**
  A:那就一直挂在他们那步,交期风险还在——催他们确认,或改选退交期。
