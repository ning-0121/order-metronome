## 0. O1 核心思想(一句话)
**物料定义只录一次进 Master,以后每单从 Master「选」而不是「敲」;本订单只填这一单特有的:单耗、颜色、位置、备注、特殊要求。** —— 这就是"比 Excel 快"的根本机制(选 + 复用 + 智能默认),O1 不靠 AI。

## 1. Material Master 数据模型(🆕 公司级唯一物料主数据)

`material_master`(全公司共享,SAP 式 Material Master):
| 字段 | 说明 |
|---|---|
| id / material_code(唯一) | 物料编码,如 FAB-0001 / TRM-0012(可自动生成)|
| material_name | 物料名(主面料/拉链/吊牌…)|
| category | 供应链分类(fabric/trim/packing/print/washing/embroidery/service/other)= V2.1 枚举 |
| default_unit | 默认单位(kg/pcs/m/yard…)|
| default_consumption | 默认单耗(可空;每单可覆盖)|
| default_supplier_id | 默认供应商(→ factories)|
| default_lead_days | 默认交期(可空;喂 B1 的 supplier_profile 来源,优先级见 V2.1 §8b)|
| specification | 规格(成分/克重/纱支…)|
| default_loss_rate | 默认损耗%(可空)|
| image_url | 物料图(证据,不参与计算)|
| status | active / archived |
| usage_count | 被多少订单用过(排序用)|
| created_by / created_at / updated_* | 审计 |

> 价格历史**不放这里**:复用现有 `price_history`(按 material 维度查),Master 只存"默认/参考"。

## 2. Material Master 如何从现有数据沉淀(不凭空建)
**一次性回填 + 持续沉淀,绝不让业务从零建库:**
- **种子来源(回填)**:
  1. `customer_trim_library`(已有客户辅料库,含 trim_name/spec/supplier/lead)→ 去重导入 Master(辅料/包材为主)。
  2. `quoter_fabric_records`(报价库面料:material_code/单价/交期)→ 导入 Master(面料为主)。
  3. 历史 `materials_bom`(若有)→ 去重补充。
- **持续沉淀(组织化生长)**:业务在录入时新建的物料 → **一键"加入主数据"** → 下次任何订单可复用。Master 越用越全。
- **去重键**:material_name + category(+ spec 可选),避免重复条目。

> 注:历史"原辅料单文件"的 AI 解析沉淀留到 O6(O1 不做 AI)。O1 的种子来自上面 3 个**已有结构化表**。

## 4. 如何让录入比 Excel 更快(O1 不靠 AI,靠这 5 招)
1. **选,不敲**:从 Master 搜索选物料 → 名称/类别/规格/默认供应商/默认单耗**自动带出**,只补单耗/颜色。
2. **客户常用库带入**:复用现有 `importFromTrimLibrary`(已能按客户品牌一键带入多条)→ 秒填一批。
3. **复制上一单**:把某历史订单的 Material Package 整包复制过来微调(翻单场景极快)。
4. **智能默认**:单位/损耗/供应商默认从 Master 带,留空即用默认。
5. **键盘友好网格**:Tab 流转、回车下一行、粘贴多行(像 Excel 一样敲,但底层是结构化)。

> 衡量标准:一个 10 物料的订单,**从 Master 选 + 带入 + 补单耗 应 ≤ 2 分钟**,显著快于"开 Excel 排版再传"。验收要实测(§14)。

---

## 5. 录入时如何搜索 / 选择 / 复用物料
- **搜索**:输入名称/编码/类别即时过滤 Master(usage_count 高的靠前)。
- **选择**:点选 → 生成一条 Package 行(带 master_id + 默认值)。
- **复用**:① 客户常用库带入 ② 复制上一单 ③ Master 高频物料快捷区。
- **新物料**:搜不到 → "新建物料"内联表单 → 存进 Master(下次可复用)+ 同时加入本单 Package。

## 6. 订单级字段如何填(本单特有,不进 Master)
| 字段 | 必填 | 说明 |
|---|---|---|
| qty_per_piece(单耗)| ✅ | 本单该物料单件用量 → 喂 B1 MRP 的 gross |
| color(颜色)| | 本单配色 |
| placement(位置/部位)| | 大身/门襟/袖口… |
| unit(单位)| | 默认来自 Master,可覆盖 |
| supplier(供应商)| | 默认来自 Master,可覆盖 |
| remarks / special_requirements(备注/特殊要求)| | 本单特殊工艺/要求 |

> 物料**定义**(名/编码/类别/规格)来自 Master(只读或受控覆盖);**用量与本单属性**在订单填。

## 7. 与 materials_bom 的关系(演进,不新建第二套)
**`materials_bom` 就是 Material Package 的明细行**,O1 在其上:
- **加列**:`material_master_id`(FK→material_master,可空=临时物料)、`special_requirements`、(可选)`category`(从 Master 带,便于 B1 直接用)。
- **保留**:现有 material_name/material_type/qty_per_piece/color/placement/spec/supplier + Step A 的 submit_status/version。
- B0/B1 读 materials_bom **不变**(只是现在有真实数据 + master 链接)。

## 8. 与 B1 / material_requirements 的关系
- Material Package(materials_bom)→ 提交采购(B1 已建)→ 冻 Snapshot → MRP → material_requirements。**O1 让 materials_bom 真正有数据,B1 立刻活。**
- **category 衔接**:B1 现在用 `material_type→category` 映射;O1 后 category 应直接来自 Master。**O1b 实现时一个小衔接**:Snapshot/MRP 优先读 Package 的 category(来自 Master),回退到 material_type 映射 —— 保证 B1 拿到精确类别(此衔接在 O1b 写,O1 设计只标明,不改 B1 现状)。
- 单耗(qty_per_piece)→ B1 gross;默认供应商/交期 → B1 lead 的 supplier_profile 来源(优先级见 V2.1 §8b,B2 落地)。

## 9. 与 Attachment / Evidence 的关系
- 客户给的**原辅料单文件(trims_sheet)仍上传为附件 = 证据**,**不是数据源**。
- 业务录入时可**对照**附件填结构化 Package;真相永远是结构化 Package,不是文件。
- 文件 AI 解析→预填 = O6,不在 O1。O1 里文件就是"放着看"的证据。

## 10. 权限 / 版本 / 审批 / 提交采购流程
- **Material Package 录入**:业务/理单(数据归属:业务拥有物料定义)。
- **Material Master 维护**:新建可由业务录入时顺手加;**编辑/归档已有 Master** 受控(理单/管理员),防止改坏共享数据。
- **版本/审批**:复用 B0/B1 —— 提交采购时冻 Snapshot(版本),`materials_bom.version` 管修订;**"提交采购"本身就是审批闸**(提交即锁 BOM、生成需求)。O1 不另造审批流。
- **提交采购**:复用已上线的 `submitBomToProcurement`(B1)。O1 只把上游数据做实。

## 11. Migration Strategy(设计层)
- **Evolution**:新增 `material_master`;`materials_bom` 加 `material_master_id`/`special_requirements`/(可选 category)。其余全留。
- **种子回填**:一次性脚本从 `customer_trim_library` + `quoter_fabric_records` 去重导入 Master(人工执行 + 可 dry-run 预览)。
- **Feature Flag**:新 Material Package 录入与现有 BomTab 并行,验证后切换。
- B0/B1/采购中心/ProcurementTab **零改动**。
- SQL 人工执行 / build+check / diff 审 / 批了才 push。

## 14. 验收标准
1. `material_master` 建好,种子从 customer_trim_library + quoter_fabric_records 去重导入(数量合理、无明显重复)。

3. 客户常用库带入、复制上一单 都可用。
4. 新建物料能存进 Master 并立即可复用。
5. `materials_bom` 行带 `material_master_id` + 单耗;**提交采购(B1)能正常冻快照 + 生成 material_requirements**,数值正确。
6. **速度实测**:10 物料订单从选到提交 ≤ 2 分钟,且业务主观"比开 Excel 快"。
7. 现有采购中心 / ProcurementTab / B0 / B1 零影响。
