# 外贸服装采购/生产系统 · 对标设计审计与借鉴清单

> 视角:顶级外贸服装生产采购系统设计师。把 QIMO OS 对标全球最佳实践,找**可优化 / 可借鉴**的功能。
> 立场:你们的差异化(**卡风险不走流程 + 交付置信度引擎 + 归并/补采购/收货闸这套采购执行**)业界稀缺且领先。
> 建议**在 DNA 上补高杠杆的几块**,不是变成臃肿 ERP。遵循 DP-7 分阶段、系统计算·人决策、单一真相源。

---

## 一、结论先行

你已经强在:**采购执行层**(归并 netting、补采购+财务闸、收货批次+±10%闸、库存抵扣+预留、按款按色单耗)、**交付风控**(18 关卡 + Runtime 交付置信度)、**Quote→PO→Order 派生链**、**AI 录入**(PO 解析/小绮)。这几块比多数外贸 ERP 更"活"。

**最该借鉴的 3 大簇**(按杠杆排序):
1. **供应商智能**——你有供应商主数据,但缺"供应商表现"这层真相:交期达成率(OTD)、质量不良率(PPM)、价格竞争力指数、产能负荷日历、合规/验厂状态。**这是把"采购执行"升级成"采购决策"的关键**。
2. **产前物料确认工作流**——你有产前样(款级),但缺**物料级**的对色/缸差/辅料卡确认(lab dip / strike-off / trim card approval)。外贸大货最常翻车就在"色没对好就下大货"。
3. **落地成本 + 合规追溯**——你有成本核算/汇兑,但缺 Incoterm 感知的**落地成本(含关税/运保)** 和 **验厂/认证/供应链追溯**(BSCI/Sedex/GOTS/OEKO-TEX/GRS/DPP)。这是拿欧美大客户的门票。

外加一块横向的:**客户可视门户**(买家自助看订单进度/文件),多数领先 sourcing 平台的标配。

## 三、借鉴清单(分优先级,每条:是什么 / 谁做得好 / 为何适合你 / 接在哪 / 粗估)

### T1 · 高杠杆,最该先做

**T1-1 供应商表现记分卡(Supplier Scorecard)**
- **是什么**:每个供应商自动累计 OTD(交期达成率)、质量不良率(收货 QC 判定/让步/拒收派生)、价格竞争力(vs 历史中位/建议价)、响应速度、补采购频次;综合评级 A/B/C。
- **谁做得好**:Inspectorio、TradeBeyond(CBX)、Fashinza 的供应商绩效面板。
- **为何适合你**:你已有全部原料——收货批次、±10% 判定、让步/拒收、下单/催货时间、底价 vs 建议价。**纯派生,不用人录**,正合"系统计算·人决策"。让"下给谁"从拍脑袋变成有据。
- **接在哪**:suppliers + procurement_line_items/goods_receipts 派生;供应商详情页加记分卡 tab。
- **粗估**:中(派生计算 + 一个面板)。

**T1-2 产前物料确认工作流(Lab Dip / Strike-off / Trim Card)**
- **是什么**:大货前对每个物料(面料对色 lab dip、印花 strike-off、辅料 trim card)走"寄样→客户/内部确认→通过/重做"状态机,**未全部通过 → 卡住大货采购/生产闸**。
- **谁做得好**:Centric PLM、Bamboo Rose、Surefront 的 material approval。
- **为何适合你**:完美契合"卡风险不走流程"——把"色没对好就下大货"这个外贸头号翻车点变成硬闸。你已有产前样(款级)+ 多方确认机制,复用即可下沉到物料级。
- **接在哪**:materials_bom 上挂 approval 状态;里程碑加"物料确认"闸,联动采购下单闸。
- **粗估**:中(状态机 + 闸 + 寄样记录)。

**T1-3 落地成本(Landed Cost, Incoterm 感知)**
- **是什么**:成本 = 面料+辅料+CMT+损耗 + **运保费 + 关税/退税 + 佣金**,按 Incoterm(FOB/CIF/DDP)算到岸/落地成本与真实毛利。
- **谁做得好**:BlueCherry、Infor Fashion、TradeBeyond 的 landed cost。
- **为何适合你**:你有成本基线 + 财务汇兑 + 退税(finance 有 tax-refund),差的是把这些**按 Incoterm 归集成一张落地成本**。让报价/决算的毛利是"真"毛利。
- **接在哪**:profit.service + 财务 budget/settlement;订单成本页加 Incoterm + 落地成本视图。
- **粗估**:中(口径 + 跨 QIMO/财务取数,注意刚审出的汇率/吞零问题)。

### T2 · 中杠杆,拉开与同行差距

**T2-1 多供应商 RFQ / 结构化比价**
- **是什么**:一个物料需求 → 发给 N 个供应商询价 → 收报价 → 结构化比价(价/期/MOQ/账期/评级)→ 一键选定建 PO。
- **谁做得好**:Fashinza、TradeBeyond、SAP Ariba(通用)。
- **为何适合你**:你有归并 + 上次采购价记忆,再进一步就是"比价选源"。补齐"寻源"这块最弱项。
- **接在哪**:procurement_items 需求 → RFQ 对象 → 报价 → 选定回填 PO。

**T2-2 供应商产能/订舱日历(Capacity Booking)**
- **是什么**:供应商/工厂的产能按周负荷可视,下单前"订产能位",避免旺季压单、交期落空。
- **谁做得好**:Coats Digital FastReactPlan、Exenta 的排产/负荷。
- **为何适合你**:直接喂交付置信度引擎——"这家这周已满"应该是扣分/预警因子。
- **接在哪**:供应商 + 交付置信度;新增 capacity 维度。

**T2-3 AQL 验货 + 缺陷代码 + CAP 整改闭环**
- **是什么**:中查/尾查按 AQL 抽样表(样本量/接收数)判定,缺陷按标准代码归类,不合格触发 CAP(整改→复验)闭环;面料 4 分制验布。
- **谁做得好**:Inspectorio、QIMA。
- **为何适合你**:你有 QC 关卡 + 让步/拒收,升级成"标准化 + 可追溯 + 整改闭环",且喂供应商记分卡(T1-1)。
- **接在哪**:QC 关卡 + goods_receipts;缺陷代码库。

**T2-4 单证自动生成(PI/CI/PL/CO)+ HS/关税 + 付款里程碑/LC**
- **是什么**:从订单/发货数据一键出 PI/CI/装箱单/产地证;HS 编码 + 关税;定金/尾款/LC 付款节点跟踪。
- **谁做得好**:TradeBeyond、Zedonk、外贸单证类 SaaS。
- **为何适合你**:你有文件中心 + 富录入(双语描述/箱数),差临门一脚的单证模板引擎。财务已有 AR/AP,补 LC/付款里程碑。
- **接在哪**:文件中心 + order_line_items + 财务。

### T3 · 长期壁垒 / 大客户门票

**T3-1 合规 + 供应链追溯(验厂/认证/DPP)**
- **是什么**:供应商验厂(BSCI/Sedex/WRAP)有效期跟踪、材料认证(GOTS/OEKO-TEX/GRS)、成分链路追溯(chain of custody)、数字产品护照(DPP,欧盟 2027 起强制)。
- **谁做得好**:Sourcemap、TextileGenesis、TrusTrace、Inspectorio Rise。
- **为何适合你**:欧美大客户的准入门槛,也是十年布局(DP-6)。先从"验厂到期提醒 + 认证附件"轻量起步。

**T3-2 客户可视门户(Buyer Portal)**
- **是什么**:买家自助登录看自己订单的进度(你的 18 关卡/置信度直接复用)、下载 PI/文件、确认 lab dip/产前样。
- **谁做得好**:TradeBeyond、Surefront、多数现代 sourcing 平台。
- **为何适合你**:你的交付置信度卡是天然对客户展示物,极强的差异化体验。注意审计出的越权红线——门户必须严格租户隔离。

**T3-3 外发单 / 委托加工工序 WIP**
- **是什么**:委托加工(外发)订单类型 + CMT 工序级 WIP(裁/车/后整)。
- **为何适合你**:memory 里已规划"外发单",这是补全订单类型 + 生产执行颗粒度。

## 四、不建议照搬(避免臃肿,守住 DNA)
- **重型排产/APS 全套**(每工序秒级排):你是贸易+发外为主,不必做工厂级 MES/APS,capacity 日历(T2-2)够用。
- **全功能 PLM**(3D/款式开发/趋势):你不是品牌方,tech pack/尺寸表做到"够生产用"即可,别陷入设计端。
- **流程审批全家桶**:违背"卡风险不走流程"。新功能一律问"这是卡风险还是走流程",只做前者。
- **AI 拥有真相**:预测/自动建议可以,但采购/成本决策的最终真相永远人定(DP-5)。

---

## 五、落地建议(80/20)
1. **先做 T1-1 供应商记分卡**——纯派生、投入中、直接把采购从执行升级为决策,且喂后续所有供应商决策。
2. **紧接 T1-2 产前物料确认闸**——最契合 DNA、最能防大货翻车。
3. **T1-3 落地成本**——但先修审计出的利润算账 bug(汇率硬编码/`||`吞零)再叠加,别在错地基上盖楼。
4. T2/T3 按客户诉求排:接欧美大客户就提前 T3-1 合规;买家要可视就 T3-2 门户。
