# 考核制度设计 V1.0 —— 录入积极性 × 交付执行力

> 2026-07-14 拟。目的:让「系统即真相」——每个人及时把自己那块录进系统,交付风险才看得见、才卡得住。
> 考核不是为了扣钱,是把「及时录入 + 按时干活」变成**看得见、有回报**的事。

## 1. 问题
录入不积极 → 数据滞后 → 风险面板全是虚红、真风险被淹 → 管理失效。
根子:录入没好处、不录也没坏处。考核 = 给录入装正反馈。

## 2. 对象 & 范围
- **对象**:各执行角色对**自己名下的节点**(业务执行 / 跟单 / 采购 / 生产QC / 物流 / 财务)。主管另设团队考核。
- **范围(公平基线)**:**2026-07-13(本周一)起**的节点才考核 = 进行中订单的本周及以后节点 + 本周起新建订单全部节点。**之前一律不追溯**(已进算法)。
- **周期**:周(即时反馈、出榜)+ 月(汇总、挂钩激励)。

## 3. 考核维度(4 维,都指向「及时录入」)
| 维度 | 权重 | 衡量 |
|---|---|---|
| 准时率 | 40% | 名下节点在截止日当天或之前完成/更新的比例 |
| 响应速度 | 30% | 节点「进行中→完成」平均天数 |
| 零逾期 | 20% | 当前手头有无逾期未处理节点 |
| 闭环质量 | 10% | 多方确认及时、无被上报升级、证据/字段填全 |

综合 0-100,沿用现有引擎加权(可调)。

## 4. 等级 & 标准线
| 等级 | 分数 | 处置 |
|---|---|---|
| S | ≥90 | 红榜 + 奖励 |
| A | 75-89 | 良好达标 |
| B | 60-74 | 及格线 |
| C | 40-59 | 预警·黄牌·主管提醒 |
| D | <40 | 黑榜·约谈 |

**红线**(触发即预警,无论总分):当前逾期节点 ≥ 3;月逾期率 > 30%。

## 5. 激励(正向为主)
- **正向**:月度红榜前3公示;连续达标(streak)奖金系数上浮;部门冠军;及时录入积分季度兑换。
- **约束**:月度 D 级/黑榜→约谈 + 挂钩绩效奖金;连续预警→影响调薪/晋升。
- **团队**:**部门平均分纳入主管考核**——主管对本部门录入负责,不只压员工。

## 6. 公平机制(否则考核变对抗)
- 基线不追溯(已做);
- 工作量归一:按名下节点数加权;
- 外部原因豁免:客户改单/外部延误经延期申请批准后不扣分;
- 申诉通道:对判定有异议可申诉,主管/admin 复核改判留痕。

## 7. 落地节奏(别一上来扣钱)
- 第 1 周(本周·试运行):只出榜、不挂钩,让大家看到数据、习惯录入;
- 第 2-4 周:正式挂钩月度激励;
- 每月复盘调权重/红线/额度。

## 8. 系统侧待建(用户点单)
A 看板加基线横幅 + 红黑榜 + 达标线 · B 周报/月报自动推送 · C streak/积分 · D 申诉入口 · E 部门主管视图。

## 9. 待用户拍板(管理决策,非技术)
权重 / 红线阈值 / 激励额度 / 挂钩力度(试运行几周、扣钱与否)。
