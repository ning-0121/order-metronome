## 一、产品定位

**订单节拍器**是一套专为服装供应链设计的全链路订单管理系统。核心理念：**卡风险，而不是走流程**。

它不是简单的任务清单，而是一个"订单指挥中心"：

- 每一笔订单的 30 个生产节点，全部有截止日期、负责人、完成标准
- AI 实时扫描每个节点的状态，主动发现交期风险、资料缺失、成本超标
- 不同角色看到不同视图，销售只看客户维度，跟单只看生产维度，财务只看付款维度

## 三、用户角色与权限

系统支持 **7 种角色**，每个账号可同时持有多个角色：

| 角色 | 英文 key | 核心职责 |
|------|----------|---------|
| 业务 | sales | 客户沟通、PO 确认、报告提交 |
| 跟单 | merchandiser | 生产跟进、里程碑提交、日报 |
| 采购 | procurement | 原料采购、采购审批 |
| 生产 | production | 工厂管理、生产报告 |
| 品控 | qc | 质量检验、QC 报告 |
| 物流 | logistics | 订舱、报关、出运 |
| 财务 | finance | 付款审批、成本管理 |
| 管理员 | admin | 全系统权限（白名单：alex / su） |

**权限分层规则：**
- 财务 / 管理员 / 助理管理员 / 生产经理：可查看所有订单
- 其他角色：仅看到自己负责的订单（`owner_user_id` 匹配）
- 敏感价格信息（利润率、FOB/DDP 单价）：仅销售、财务、管理员可见
- 延期申请：跟单和销售均可发起，管理员审批

## 四、核心功能模块

### 4.1 订单管理

**订单主表（`orders`）核心字段：**
- 订单号（自动按年份+序号生成，如 QM-2026-001）
- 客户名称、工厂名称、款式描述
- 下单日期、工厂开工日期、ETD（预计离港）/ ETA（预计到港）
- 贸易条款（FOB / DDP）
- 数量、颜色数、尺码范围
- 生命周期状态：active / sample-only / completed / cancelled

**订单列表页：**
- 支持按关键字、状态、角色、到期时间筛选
- 显示每个订单的当前节点进度条（0%～100%）
- 高亮显示逾期订单（红色）、今日到期订单（橙色）

### 4.2 18 关卡里程碑系统

**标准生产订单：30 个节点（分 4 个阶段）**

#### 阶段 A：订单启动（7 关）
| # | 节点名称 | 负责角色 | 说明 |
|---|---------|---------|------|
| A1 | PO 确认 | 业务 | 客户正式 PO 上传确认 |
| A2 | 财务审批 | 财务 | 订单利润率/付款条件审批 |
| A3 | 订单资料 | 跟单 | 款式单、BOM、大货材料确认 |
| A4 | 采购单 | 采购 | 原料采购单制作完成 |
| A5 | 采购审批 | 财务 | 采购预算审批 |
| A6 | 采购下单 | 采购 | 正式向供应商下单 |
| A7 | 原料检验 | 品控 | 到料检验合格 |

#### 阶段 B：产前样（4 关）
| # | 节点名称 | 负责角色 | 说明 |
|---|---------|---------|------|
| B1 | 产前样完成 | 生产 | 工厂产前样制作完成 |
| B2 | 产前样寄出 | 跟单 | 寄给客户或业务 |
| B3 | 产前样确认 | 业务 | 客户确认产前样 |
| B4 | 大货启动 | 生产 | 正式进入大货生产 |

#### 阶段 C：生产出货（5 关 + 更多子节点）
| # | 节点名称 | 负责角色 | 说明 |
|---|---------|---------|------|
| C1 | 中期验货 | 品控 | 生产 50% 时现场验货 |
| C2 | 尾期验货 | 品控 | 生产 90% 时最终验货 |
| C3 | 包装到位 | 跟单 | 包材、吊牌全部到厂 |
| C4 | QC 预约 | 品控 | 约定第三方 QC 时间 |
| C5 | QC 完成 | 品控 | QC 报告通过 |

#### 阶段 D：出运（2 关）
| # | 节点名称 | 负责角色 | 说明 |
|---|---------|---------|------|
| D1 | 订舱完成 | 物流 | 船期确认，货代确认 |
| D2 | 出运完成 | 物流 | 货物装船，提单完成 |

**节点状态流转：** `pending` → `in_progress` → `completed` / `skipped` / `blocked`

**每个节点支持：**
- 截止日期（系统自动计算，基于工厂开工日 + 比例）
- 负责人指派
- 完成条件 checklist（多项可选）
- 附件上传（图片、PDF、Excel、视频）
- 延期申请（含原因分类、影响评估）
- 操作审计日志（记录谁在什么时间做了什么）

### 4.3 生产进度模块

**三种报告类型：**

1. **日报（daily_report）**：每日生产数量、工序进度、问题记录
2. **品质报告（quality_report）**：缺陷率、缺陷类型、图片证据
3. **专项报告（special_report）**：里程碑节点关联的专项进度说明

**数据结构（`production_reports`）：**
- `qty_produced`：今日产量
- `qty_cumulative`：累计产量（系统自动计算，防手动错误）
- `defect_rate`：缺陷率（数据库 GENERATED ALWAYS AS 自动计算列：缺陷数 / 总件数）
- `report_subtype`：报告子类型
- `notes`：备注
- 关联里程碑节点

**权限控制：**
- 跟单（merchandiser）/ 生产（production）：可填写实到数量
- 其他角色：只读

### 4.7 财务与成本控制

**成本基准（`order_cost_baseline`）：**
- 面料成本 / CMT 费用 / 其他成本
- FOB 单价 / DDP 单价
- 汇率（实时获取）
- 利润率（自动计算）

**利润率预警：**
- 🔴 红色：低于 8%（高风险，需财务审批）
- 🟡 黄色：8%～15%（关注）
- 🟢 绿色：高于 15%（健康）

**成本超标触发器（自动）：**
- 面料采购成本超预算 → 自动通知财务 + 相关管理层
- CMT 超出估算 → 自动预警

**付款管理（`order_financials`）：**
- 订金状态（已付 / 待付）
- 尾款状态（已付 / 待付）
- 是否允许生产（财务开关）
- 是否允许出运（财务开关）
- 付款锁定（超期未付款，冻结订单操作）

### 4.10 订单时间轴引擎

**核心算法：**
- 基准周期：45 天（大货）/ 14 天（打样）
- 每个节点有固定的"百分比比例"（如中期验货 = 67% 进度）
- 系统根据工厂开工日和 ETD 计算实际可用天数
- 按比例缩放所有节点的截止日期
- 考虑中国法定节假日（2025-2027 已内置）
- DDP 贸易条款：ETA - 25 天 = 锚定日期（中美航运标准时效）
- FOB 贸易条款：以 ETD 为锚定日期

## 五、AI 能力系统

### 5.1 AI Agent（智能顾问）

Agent 是一个主动决策引擎，不需要用户触发——它持续扫描订单状态，**主动生成建议和预警**。

**9 种 Agent 动作类型：**

| 动作类型 | 说明 |
|---------|------|
| `nudge` | 提醒负责人尽快完成节点 |
| `notify` | 通知相关方（如客户确认慢，通知业务） |
| `escalate` | 升级给管理层（超期 N 天自动升级） |
| `suggest` | 给出操作建议（如"建议提前锁舱位"） |
| `warn` | 发布风险预警 |
| `block` | 标记节点为阻断状态 |
| `defer` | 建议推迟某节点（资源不足时） |
| `auto-complete` | 自动标记已满足条件的节点为完成 |
| `cross-order` | 跨订单协调（如同工厂多个订单冲突） |

**Agent 限速：**
- 每个订单每天最多触发 5 次
- 全局每小时最多 20 次
- 熔断机制：连续失败 5 次 → 暂停 1 小时

**AI Enhancement Layer：**
- Agent 建议默认用规则引擎生成
- 开启 `AGENT_FLAG_AI_ENHANCE` 后，用 Claude 为每条建议生成自然语言解释
- 使用 Anthropic Batch API 批量处理，降低成本

### 5.2 AI Skills（智能技能库）

系统集成 **7 个 AI 技能**，每个技能都有独立的缓存、熔断、审计日志。

---

#### 技能 1：风险评估（Risk Assessment）

- **模型**：Claude Sonnet 4
- **缓存**：6 小时
- **评估维度（12 项，加权）：**

| 维度 | 风险因素 |
|------|---------|
| 客户风险 | 新客户、历史投诉、付款记录不良 |
| 工厂风险 | 新工厂、多工厂混产、历史延误 |
| 产品复杂度 | 高弹面料、大码、复杂印花/绣花 |
| 颜色风险 | 浅色（污染风险）、色系冲突 |
| 订单数量 | 极大或极小（工厂不重视） |
| 时间紧迫 | 可用天数 vs 标准周期比值 |
| 包装复杂度 | 个性化包装、礼品盒、复杂折叠 |
| 资料缺失 | 关键文件未上传 |
| 跳过打样 | 新工厂但跳过产前确认 |
| 客户延误历史 | 历史订单中客户确认速度 |
| 工厂负荷 | 当前同工厂在制订单数量 |
| 旺季风险 | 春节/圣诞等高峰期重叠 |

- **输出**：0～100 综合风险分 + 各维度详情 + 证据引用 + 按角色分类的操作建议

---

#### 技能 2：缺失资料检查（Missing Info Check）

- **模型**：纯规则，无 AI 调用
- **缓存**：5 分钟
- **检查 24 项：**

**文件类：**
客户 PO、内部成本单、客户报价、生产订单、原料清单、包装要求、款式资料、中期验货报告、尾期验货报告、装箱单

**数据字段类：**
工厂指派、工厂开工日期、ETD/ETA（DDP 需要两者）、订单数量、款式/颜色数量

**确认类：**
产前样客户确认、大货材料确认

- **输出**：按"距离成为阻断的天数"排序，越紧急越靠前，严重程度随时间动态提升

---

#### 技能 3：报价审核（Quote Review）

- **模型**：Claude Sonnet 4（用于文件识别与提取）
- **缓存**：1 小时
- **能力：**
  - 自动识别并解析附件（PDF / 图片 / Excel）中的成本单、报价单、PO
  - 提取：单位成本、面料成本、CMT 费用、其他费用
  - 计算利润率，实时汇率换算（多币种支持）
  - 价格一致性检查（内部报价 vs 客户报价是否匹配）
- **输出**：结构化成本分解 + 利润率评级（红/黄/绿）+ 异常项预警

---

#### 技能 4：客户邮件洞察（Customer Email Insights）

- **模型**：Claude Sonnet 4
- **缓存**：2 小时
- **数据源**：最近 90 天客户邮件（`mail_inbox` 表）
- **分析内容：**
  - 扫描客户近期关注点
  - 发现被忽略的客户请求（按严重程度：高/中/低）
  - 生成建议回复：邮件主题 + 3 个关键点 + 语气建议
- **知识注入**：结合订单知识库（客户偏好、历史沟通模式）

---

#### 技能 5：交期可行性分析（Delivery Feasibility）

- **模型**：规则引擎 + 历史数据
- **缓存**：30 分钟
- **基准工期（历史中位数）：**
  - 大货新客/新款：50 天
  - 大货复购：35 天
  - 打样：21 天
- **分析维度：**
  - 已订天数 vs 基准工期
  - 工厂历史延误天数（从过往订单中统计）
  - 工厂当前在制订单数（正常 8-10 单，高风险 >15 单）
- **输出**：🟢 可行 / 🟡 紧张（有风险）/ 🔴 高风险，配合具体数据说明

---

#### 技能 6：今日行动清单（Today's Actions）

- **模型**：纯规则，实时聚合
- **缓存**：5 分钟
- **内容：**
  - 抓取当前用户所有进行中里程碑
  - 按紧急程度分类：逾期 / 今日到期 / 3 天内 / 正常
  - 包含待审批事项 + 未读通知
- **输出**：每个用户的"今日 Top 3 紧急事项" + 倒计时

---

#### 技能 7：AI 质检（Garment Defect Detection）

- **模型**：Claude Sonnet 4（多模态视觉分析）
- **输入**：QC 现场照片（中期验货、尾期验货、打样、面料、包装）
- **检查项：**

| 类别 | 具体检查 |
|------|---------|
| 缝制工艺 | 跳针、脱线、针头外露 |
| 面料 | 色差、挂丝、起球 |
| 印花/绣花 | 错位、模糊、渗色 |
| 辅料 | 拉链、纽扣、织带 |
| 整烫 | 褶皱、烫伤、变色 |
| 对称性 | 左右对称、领口平整 |
| 包装 | 装箱方式、折叠手法、吊牌位置 |

- **批量模式**：最多同时分析 5 张图片，合并结果，取最严重问题作为整体判定
- **输出**：缺陷清单（类型/位置/严重程度：严重/主要/次要）+ 质量评分（0-100）+ 整改建议

---

### 5.3 AI 可靠性保障

**熔断机制（Circuit Breaker）：**
- 连续失败 5 次 → 自动暂停 1 小时
- 状态记录在 `ai_skill_circuit_state` 表
- 恢复后自动重启，无需人工干预

**结果缓存：**
- 基于输入哈希值缓存
- 各技能 TTL 不同（5 分钟～6 小时）
- 订单状态变化自动失效

**影子模式（Shadow Mode）：**
- 新技能上线默认开启影子模式
- 影子模式：运行计算但不展示给用户
- 7 天后评估准确率，合格后正式启用

**全链路审计：**
- 所有 AI 技能运行记录在 `ai_skill_runs` 表
- 包含：输入哈希、输出结果、置信度分数、耗时、错误信息
- 支持离线分析 AI 决策质量

## 六、自动化系统（17 个 Cron Jobs）

### 每日自动任务（北京时间 08:00）

| 任务 | 说明 |
|------|------|
| `daily-tasks` | 向每位成员推送个人 Top 3 紧急事项（系统通知 + 可选 WeChat） |
| `daily-briefing` | 为销售团队生成邮件日报（邮件摘要、紧急标记、24 小时汇总） |
| `order-audit` | 扫描所有活跃订单：缺工厂/负责人/数量，过期记录，超时节点 |

## 九、安全与合规

**认证：**
- 仅限 @qimoclothing.com 邮箱注册登录
- 管理员账号白名单（alex / su），即使同域名其他账号也无管理员权限

**数据隔离（Row Level Security）：**
- 所有敏感表均开启 RLS
- 使用 SECURITY DEFINER 函数 `user_can_access_order()` 作为统一门控
- 跨角色数据访问需通过函数验证，不允许直接 SQL 绕过

**操作审计：**
- 每次里程碑状态变更记录完整日志（谁/什么时间/从什么状态改为什么状态）
- AI 技能运行全程记录（输入哈希/输出/耗时/错误）

**价格保护：**
- 利润率、FOB/DDP 单价仅向销售、财务、管理员角色展示
- 生产/跟单/品控等角色无法看到价格信息

```

    ↓

    ↓

    ↓
```
